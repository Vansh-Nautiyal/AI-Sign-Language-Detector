# 🤟 Real-Time ASL Sign Language Reader

A complete Python project that reads **American Sign Language (ASL) alphabets (A–Z)** in real-time using your webcam, MediaPipe hand landmarks, and a lightweight Dense Neural Network (MLP).

---

## 📁 Project Structure

```
asl_reader/
├── data/
│   └── dataset.csv          ← generated by collect_data.py
├── model/
│   ├── asl_model.h5         ← generated by train_model.py
│   ├── class_names.txt      ← generated by train_model.py
│   └── training_history.png ← accuracy/loss curves
├── src/
│   ├── collect_data.py      ← Step 1: collect hand landmark data
│   ├── train_model.py       ← Step 2: train the MLP

├── logs/

├── requirements.txt

```

## 🚀 Quick Start (3 steps)

### Step 1 — Collect training data

```bash
python src/collect_data.py
```

| Key | Action |
|-----|--------|
| `A`–`Z` | Save current frame landmarks with that label |
| `Q` | Quit collection |

**Tips for good data:**
- Collect **50–100 samples per letter** for decent accuracy
- Vary your hand distance and slight angle
- Good lighting improves landmark detection

Data is saved to `data/dataset.csv`.

---

### Step 2 — Train the model

```bash
python src/train_model.py
```

- Trains a 3-layer MLP: `63 → 128 → 64 → 26`
- Uses early stopping (patience = 10 epochs)
- Saves model to `model/asl_model.h5`
- Saves class names to `model/class_names.txt`
- Saves training curves to `model/training_history.png`

---

### Step 3 — Run the real-time app

```bash
python app.py
```

| Key | Action |
|-----|--------|
| `Q` | Quit |
| `R` | Reset smoothing buffer |

---

## 🧠 Model Architecture

```
Input (63 features: 21 landmarks × x,y,z)
  ↓
Dense(128, ReLU) + BatchNorm + Dropout(0.3)
  ↓
Dense(64, ReLU) + BatchNorm + Dropout(0.2)
  ↓
Dense(26, Softmax)  →  A–Z
```

- **Optimizer:** Adam
- **Loss:** Categorical Crossentropy
- **Input normalization:** wrist-relative (subtract landmark 0)

- **Green** = high-confidence prediction (≥ 80%)
- **Yellow** = medium confidence
- **Bounding box** drawn around detected hand
- **Smoothing**: majority vote over last 10 frames

## 📊 Features

| Feature | Detail |
|---------|--------|
| Landmark normalization | Wrist-relative (translation-invariant) |
| Confidence threshold | 80% (configurable in `predict.py`) |
| Smoothing window | 10 frames majority vote (configurable) |
| Logging | Timestamped predictions → `logs/predictions.log` |
| FPS display | Rolling 30-frame average |

---

## ⚡ Configuration

Edit the top of `src/predict.py` to change:

```python
CONFIDENCE_THRESHOLD = 0.80   # raise for fewer false positives
SMOOTHING_WINDOW     = 10     # increase for more stable output
```

## 🛠️ Troubleshooting

| Problem | Fix |
|---------|-----|
| `Model not found` | Run `train_model.py` first |
| `Dataset not found` | Run `collect_data.py` first |
| `Cannot open webcam` | Change `cv2.VideoCapture(0)` index to 1 or 2 |
| Low accuracy | Collect more samples (100+ per letter), improve lighting |
| Slow FPS | Reduce camera resolution in `app.py` |
